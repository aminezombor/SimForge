# Open Questions

## Instructions

Use this file for unresolved product, scope, research, architecture, or submission questions. Do not turn assumptions into requirements. Give each question a stable ID and link its resolution to a decision or requirement update.

## Question Register

| ID | Question | Why it matters | Owner | Needed by | Status | Resolution/decision |
| -- | -------- | -------------- | ----- | --------- | ------ | ------------------- |
| OQ-001 | What is the complete product master brief? | Requirements and scope cannot be defined without it. | Project owner | Before requirements planning | Open | Pending brief |
| OQ-002 | Which hackathon category will be selected? | Determines submission positioning and checklist completion. | Project owner | Before submission planning | Open | Pending |

Allowed statuses: Open, Researching, Awaiting owner, Resolved, Closed as unnecessary.

## Resolution Workflow

1. Capture the answer and source.
2. Record a consequential choice in `DECISIONS.md`.
3. Update affected requirements, scope, architecture, roadmap, or acceptance tests.
4. Mark the question resolved and link the resulting artifacts.
