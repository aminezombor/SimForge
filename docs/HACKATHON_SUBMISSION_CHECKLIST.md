# Hackathon Submission Checklist

## Submission Identity

- [ ] Selected category: **Developer tools** (change only through approved decision)
- [ ] Final product name and branding confirmed
- [ ] Submission deadline, form fields, and judge-access window recorded privately
- [ ] `/feedback` Codex Session ID preserved privately for the submission form

## Required Deliverables

- [ ] **Working project** - P0 acceptance tests pass with retained evidence
- [ ] **Selected hackathon category** - Developer tools entered consistently
- [ ] **Public or judge-accessible repository** - access verified in a logged-out/judge-like session
- [ ] **License and third-party notices** - Apache desktop, GPL Blender extension, dependency and asset notices complete
- [ ] **Complete README and setup instructions** - install, configure, run, test, troubleshoot, uninstall
- [ ] **Sample project or sample data** - redistributable and opens without private machine paths
- [ ] **Judge-friendly test procedure** - exact actions, expected results, duration, fallback
- [ ] **Supported platforms** - Windows 11 x64, Blender/API/provider requirements and limitations
- [ ] **Demo-ready packaged build** - installer plus portable fallback; no source rebuild required
- [ ] **Project description** - problem, users, solution, differentiation, features, architecture, limitations
- [ ] **Less-than-three-minute demo video** - public YouTube link, audio narration, no secrets
- [ ] **Explanation of how Codex and GPT-5.6 were used** - research, architecture, implementation, tests, debugging, documentation
- [ ] **Important decisions made with Codex** - concise examples linked to `DECISIONS.md`
- [ ] **`/feedback` Codex Session ID** - entered only in the submission form unless publication is explicitly desired
- [ ] **Known limitations** - honest, current, and consistent across README/report/submission
- [ ] **Post-hackathon roadmap** - V1 expansion and V2 Isaac loop

## Repository Content

- [ ] Product summary and problem statement
- [ ] Architecture overview and feature list
- [ ] Installation, developer, API-key, and secret-handling instructions
- [ ] Environment Doctor and supported-version instructions
- [ ] Demo commands/actions and expected results
- [ ] Troubleshooting and clean uninstall/cleanup
- [ ] Screenshots with sanitized paths/data
- [ ] Readiness-report and validation-results examples
- [ ] Testing evidence and acceptance summary
- [ ] Dependency/license inventory and `THIRD_PARTY_NOTICES.md`
- [ ] Codex usage log and key decisions
- [ ] Repository history has no secret, private feedback ID, or unauthorized asset

## Build and Quality Gate

- [ ] Generated primitive robot path passes
- [ ] Warehouse mobile-manipulator path passes repeatedly
- [ ] Plan Mode mutation/security tests pass
- [ ] Manual Blender edit and crash/reconnect tests pass
- [ ] Safe fix is reversible; structural fix is approval-gated
- [ ] Canonical USD moves, reopens, resolves references, and passes validation
- [ ] Clean Windows/judge account follows README successfully
- [ ] Installer and portable SHA-256 hashes published
- [ ] Dependency/security/license scans reviewed
- [ ] Sample project and release contain no absolute private paths

## Demo and Copy

- [ ] Devpost description drafted and proofread
- [ ] Concise feature summary prepared
- [ ] Technical implementation summary prepared
- [ ] Demo shot list, narration, and timing match `DEMO_SCRIPT.md`
- [ ] Deterministic pre-rehearsed provider response/fallback is ready
- [ ] Demo rehearsed under 3:00 at least three times
- [ ] Audio understandable and product activity legible at video resolution
- [ ] Codex/GPT-5.6 contribution and V2 roadmap fit final 10 seconds

## Final Owner Review

- [ ] All P0 traceability rows are Tested or Accepted, or an explicit owner-approved deferral is recorded
- [ ] P1 omissions cannot destabilize the generated path
- [ ] Known limitations and risks accepted
- [ ] Submission links tested after publication
- [ ] Final submission saved/confirmed
